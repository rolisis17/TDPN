Onboarding steps

Follow the console in order

Each step is visible on its own, but the flow is designed to stay linear: sign in, register the client, then complete the operator workflow if your role requires it.

Step 1

Wallet sign-in

Choose a wallet, request a challenge, then verify with your signature to create or refresh the session.

Step 2

Client registration

Use the signed-in session to register the client, set the path profile, and optionally provide bootstrap details.

Step 3

Operator workflow

Apply the operator role, check readiness, and approve the server workflow once the application is eligible.

Step 3

Operator workflow

This is the server lane. Sign in first, then apply the operator role, check readiness, and approve only when the session is eligible.

Operator readiness
