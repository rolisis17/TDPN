Mental map

Four ways to belong to the mesh.

GPM can stay simple on the surface while the network grows in layers: planned community access, standard paid access, reliable contributors, and backbone operators.

Use Share Earn credits Spend or upgrade

Tier 0 planned

Community Mode

Free or very low-cost access for people who knowingly help the network as a capped client, relay, and exit.

Needs
Wallet, device checks, clear consent, and safety education.
Gets
Internal credits for usage, priority, retries, or upgrade discounts.
Limits
No cash payout, no stake replacement, strict traffic and abuse guardrails.

Tier 1

Standard Access

The normal GPM path: pay or stake for access, register the device, and use stable routes without becoming a community exit.

Needs
Wallet, device registration, and required balance or stake policy.
Can do
Use the VPN, keep routes automatic, and apply for reviewed server access.
Limits
No micro-relay or micro-exit contribution until higher trust is earned.

Tier 2

Reliable Contributor

A higher-trust account for users or operators with better capacity, uptime, stake, and abuse history.

Needs
More stake, healthy device checks, capacity score, and clean reputation.
Can do
Use richer paths and opt into relay or exit beta when policy allows it.
Earns
Metered weekly rewards with holds, voids, and slashing for abuse.

Tier 3

Backbone Operator

The high-trust layer for core infrastructure, high-capacity routes, authority services, and settlement operations.

Needs
Highest stake, proven uptime, key discipline, mTLS/HTTPS, and approval.
Can do
Run authority-grade services, backbone relays, exits, and validator paths.
Duty
Carry the strongest operational, audit, and governance responsibilities.

No hidden exits

Tier 0 exit mode must be explicit, warned, explained, capped, and reversible.

Credits are not stake

Community credits buy network utility; real staking and slashing stay separate.

Abuse handling stays narrow

Track account, session, exit assignment, and abuse events without browsing logs.

Resilient first connection

Many ways in, one trust rule.

A new user should not depend on one static gateway. Official manifests, mirrors, cache, and bridge invites can help discovery, but signed freshness decides trust.

01

Signed manifest

The app verifies expiry, key policy, HTTPS posture, and trusted directory lists before registering.

02

Mirrors and cache

Alternate sources can carry the same signed material, so outages do not rewrite trust.

03

Insider bridge invite

An existing user can help an outsider discover the network, but cannot become the authority.
