# Support

## Where to Ask for Help
1. General questions and setup help: open a GitHub Discussion or issue.
2. Confirmed bugs: open a GitHub issue with reproduction details.
3. Security vulnerabilities: use `SECURITY.md` reporting path, not public issues.

## What to Include
- OS and runtime environment
- Command run
- Relevant config (redacted)
- Expected vs actual behavior
- Logs (without secrets/tokens/keys)

## Response Expectations
This is an actively developed project in beta. Response times are best effort.
